网站的起源

从编程猫到独立博客，从克隆他人模版到亲手写出自己的网站——一名初中生与代码的故事

2026年06月24日 高新炀 网站 1197字 4分钟
更新于 2026年06月24日

缘起：编程猫时代

我是一名刚刚初中毕业的学生，与编程的缘分，始于小学时偶然接触到的编程猫——一个类似 Scratch 的图形化编程平台。那时我还不太懂什么叫代码，只觉得把积木块拖来拖去就能让角色动起来，像变魔法一样神奇。

从好奇到沉迷，我在编程猫上陆续做了几款小游戏，没想到其中有两三个还挺受欢迎，甚至被官方推上了精品推荐位。每次看到评论区有人说“好玩”，那种兴奋感至今还记得。

不过后来小升初，学业压力骤增，几乎没有任何空闲时间再去维护账号。纠结了很久，最后还是痛心地把编程猫账号注销了。现在回想起来，那些作品和评论都随着一次点击消失了，还是有些遗憾——那大概是我第一次体会到“创作”和“告别”之间的拉扯。

星火：遇见独立博客

真正点燃我做个人网站念头的，是在初二左右。某天我偶然点进一个独立博客，页面很简洁，内容却很打动人。那是一个用 Markdown 构建的静态博客，虽然技术不复杂，但那种“我写的东西可以这样被看到”的感觉，让我一下子被吸引了。

我也想拥有一个自己的地方，不需要多华丽，只要能安放文字和想法就行。

于是我开始四处搜索教程，费了好大功夫注册了 GitHub，找到那个博客的源码克隆下来，换上自己的文章。就这么简陋地跑了起来，心里还是挺开心的。但没过几周，那种“用别人模板”的别扭感越来越强烈——我决定，自己写。

破土：从一行欢迎语开始

当时我已经自学过一点 Python，对编程的基本逻辑还算熟悉，所以学起 HTML 和 JavaScript 并没有想象中那么吃力。花了一个下午，我写出了第一个 index.html——整个网站就只有一个页面，上面一个大大的“欢迎”，除此以外什么都没有。

虽然简陋得可怜，但那是我第一次感受到“从零开始创造一个属于自己的东西”的快乐。

生长：漫长的迭代之路

后来就是漫长的迭代。

我学会了把 HTML 拆成独立的 CSS 和 JS 文件，页面结构变清晰了；然后是功能越加越多，代码膨胀到几千行；接着学会了把 JS 拆成多个模块，又学会了建文件夹管理文件，根目录终于变得干净了。

再后来，我开始拆分 CSS，新增了多个页面，增加了导航栏，网站慢慢像点样子了。

有一段时间我把它改成了 SPA 模式（单页应用），页面切换确实很流畅，但后来了解到 SPA 对 SEO 不太友好，又折腾着改回正常的多页模式。最近几个月，我学会了用 AJAX 实现无刷新导航，也逐步把 JS 写得更加模块化，网站的体验和代码质量都提升了不少。

此刻

现在，这个网站有了文章页、作品集、友链、评论区，甚至还有一个带加载动画的炫酷进入效果。每次打开它，看到那些曾经只有“欢迎”两个字的空白页逐渐被内容填满，心里都会有很强的成就感。

它不完美，代码可能也不优雅，但它是从一腔热血和无数个尝试错误的夜晚里长出来的。就像我在某篇文章里写过的那样——代码是逻辑的诗，文字是感性的代码。这个网站，大概就是这两种东西交汇的地方。
